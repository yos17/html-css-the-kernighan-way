2. Validation States

Type in the fields, then move away. :user-valid and :user-invalid only activate after interaction — not on page load.

✕Please enter a valid email address.
not interacted
✕Must be a valid URL starting with https://
✕3–16 characters: lowercase letters, numbers, underscores only.
3–16 chars: a-z, 0-9, underscore
✕Enter a number between 1 and 100. 
/* :user-invalid — only after the user has interacted */
.form-input:user-invalid {
  border-color: #dc2626;
  box-shadow: 0 0 0 3px rgba(220, 38, 38, 0.15);
}

/* Show sibling error message using ~ combinator */
.form-input:user-invalid ~ .form-error {
  display: flex;
}

3. Floating Labels

Click a field to see the label float to the top. Uses :placeholder-shown — no JavaScript. The label must appear after the input in the HTML for the ~ sibling selector to work. 

/* :placeholder-shown = input is empty (placeholder visible) */
/* When NOT placeholder-shown = input has content → float label */
.form-input:not(:placeholder-shown) ~ .form-label,
.form-input:focus ~ .form-label {
  top: 0.25rem;
  font-size: 0.7rem;
  color: #a8d8ea;
}
/* Input needs placeholder=" " (space) for :placeholder-shown to work */

5. Password Input with Strength Indicator

The strength bar is driven by a data-strength attribute — CSS does the coloring, JavaScript only sets the number.

Min. 8 chars with uppercase, number, and symbol. 
/* CSS reads data-strength attribute to color segments */
[data-strength="1"] .strength-segment:nth-child(1)    { background: #dc2626; }
[data-strength="2"] .strength-segment:nth-child(-n+2) { background: #ea580c; }
[data-strength="3"] .strength-segment:nth-child(-n+3) { background: #ca8a04; }

6. Search with Clear Button

The × button only appears when the field has content — detected via :not(:placeholder-shown). No JavaScript for the visibility toggle. 

/* Clear button hidden when field is empty */
.input-clear { opacity: 0; pointer-events: none; }

/* Appears when placeholder is hidden (field has content) */
.search-field:not(:placeholder-shown) ~ .input-clear {
  opacity: 1; pointer-events: auto;
}

8. Keyboard Navigation

Press Tab to move through these fields. Focus order matches DOM order — never use positive tabindex values.

Tab through the fields below to see focus tracking. 
/* Natural tab order = DOM order. Never override with tabindex="N" */
input     { /* focusable by default */ }
button    { /* focusable by default */ }
[tabindex="0"]  { /* include non-focusable element in tab order */ }
[tabindex="-1"] { /* focusable by script only — not by Tab key */ }
